# Задание 2. Swagger UI

## Авторизация и использование токена

После успешной авторизации пользователя сервер возвращает JWT access-token,
который используется для выполнения последующих защищённых запросов API.
Токен передаётся в заголовках HTTP-запроса в следующем формате:

Authorization: Bearer <access_token>

---

## Выполнение запросов от имени пользователя

Все запросы, выполняемые с использованием access-token, возвращают данные,
относящиеся к тому пользователю, от имени которого выполнена авторизация.
Например, это может быть информация о назначенных пользователю курсах,
а также структура (карта) конкретного курса.

Демонстрация выполнения запросов через Swagger UI представлена в видеофайле
`swagger.mp4`.

---

## Ограничение доступа к защищённым ресурсам

При попытке получить доступ к защищённой части API, такой как получение списка
всех организаций, пользователь с ролью **студент** получает ответ сервера
со статусом **403 Forbidden**.

В ответе сервера указано, что доступ к данной информации разрешён только
пользователям с ролью **администратор**, что соответствует ролевой модели
разграничения доступа в системе.

Данное поведение подтверждает корректную работу механизмов авторизации
и контроля доступа API.
